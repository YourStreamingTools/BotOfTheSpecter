# BotOfTheSpecter Discord Bot - Version 6.2 (2026-07-03)

## Version Information

* Updated version from 6.1 to 6.2
* Focused on Twitch schedule integration, stream schedule embed improvements, and timezone conversion updates

## New Features & Major Changes

### Next Twitch Stream on Schedule Embed

* Integrates Twitch Helix **Get Channel Stream Schedule** (`GET https://api.twitch.tv/helix/schedule`)
* Adds a **Next Twitch Stream** field to the main schedule embed when posting from the dashboard
* Resolves the broadcaster via `twitch_user_id` from the `users` table (using the channel API key) when posting a new schedule message
* Schedule embed title continues to link to `https://www.twitch.tv/{USERNAME}/schedule`
* Displays when an upcoming segment exists:
  * Stream title
  * Category / game name
  * Start and end time converted from Twitch UTC (`start_time` / `end_time` RFC3339) into the user's configured schedule timezone (e.g. AEST)
  * Example format: `Friday, 4 July 2026 at 08:00 PM AEST – 10:00 PM AEST`
* Uses explicit local timezone strings rather than Discord `<t:...>` timestamps so all viewers see the streamer's configured timezone

### Stream Schedule Resume on Bot Restart

* On startup, loads all saved rows from `stream_schedule_messages`
* Edits the existing schedule embed in Discord instead of requiring a dashboard re-post
* Re-fetches Twitch schedule data and updates the **Next Twitch Stream** field (or removes it if no data)
* Resolves Twitch user info on restart via `channel_mappings` (`twitch_user_id`, `username` by `guild_id`)
* Continues resuming the separate per-minute timezone conversion message updates
* Shared embed builder used for both new posts and restart refreshes

### Timezone Conversions

* Added **Sydney** (`Australia/Sydney`) to the live timezone conversion embed
* Sydney appears alongside UTC, US East, US Central, US Mountain, US West, London, and Tokyo when the base timezone is not Australian
* Added `AEDT` alias mapping to `Australia/Sydney` alongside existing `AEST` support in schedule and conversion helpers

## Bug Fixes & Improvements

* **Next stream now shows the correct upcoming segment**
  * Previously the first segment returned by the Twitch API could be shown even if it had already started earlier the same day. The bot now only displays segments whose `start_time` is still in the future, skips canceled segments (`canceled_until`), and scans up to 25 returned segments to find the true next stream.
* **Next Twitch Stream field hidden when there is nothing to show**
  * The field is omitted entirely when the API returns an error, no segments, only canceled segments, or vacation with no post-vacation stream scheduled. No placeholder text is displayed.
* **Twitch app token source corrected**
  * Schedule API calls now use the app access token from `website.bot_chat_token.twitch_oauth_api_token`. `get_bot_access_token()` was updated to read only that column, with `Client-Id` from the `CLIENT_ID` environment variable.
* **Schedule embed refreshes on bot restart**
  * Saved schedule messages are edited on startup so the next-stream field stays current without re-posting from the dashboard.

## Technical Details

* Added `_fetch_next_twitch_stream_info()`, `_get_next_upcoming_segment()`, `_format_local_schedule_time()`, and `_format_twitch_schedule_segment()` to the `ServerManagement` cog for Helix schedule parsing and display.
* Added `_build_stream_schedule_embed()`, `_refresh_saved_stream_schedule()`, and `_get_twitch_info_for_server()` for shared embed building and restart refresh.
* Expanded `_resume_stream_schedule_updates()` to refresh the main schedule embed, not just the per-minute timezone conversion message.
